AgroLink: AI-Driven FinTech Platform for Agricultural Micro-Investment

Ongoing SDGP (System Development Group Project)

Overview

AgroLink is a cross-platform FinTech solution bridging the financial gap between small-scale farmers and investors in Sri Lanka. It leverages an AI-powered portfolio generator and a proprietary Farmer Trust Score to optimize returns and quantify credibility. The system includes both a modern mobile app for investors and a USSD/SMS interface for farmers in low-connectivity areas.

Technologies Used

  • Frontend: React.js (Web Dashboard), React Native (Mobile App), USSD Gateway integration
  • Backend: Java Spring Boot (Microservices), Python (AI Prediction Service)
  • Database: MongoDB Atlas (NoSQL)
  • Infrastructure: AWS / Render, CI/CD Pipelines
  • Design & Modeling: Figma (UI/UX), UML (System Architecture), Draw.io

Key Skills Gained

  • System Architecture: Designed a scalable Three-Tier Microservices Architecture separating core logic, AI processing, and authentication
  • Requirements Engineering: Conducted interviews, surveys, and Stakeholder Analysis using the Onion Model
  • Ethical Computing: Ensured compliance with Sri Lanka’s PDPA (2022) and implemented Explainable AI (XAI) to mitigate bias
  • FinTech Logic: Developed algorithms for dynamic risk assessment and automated profit-sharing models

Challenges & Problem-Solving

  • The Digital Divide: Many rural farmers do not possess smartphones. USSD/SMS workflow allows feature phone users to register, receive funding alerts, and update crop status.
  • Investor Trust Deficit: Conceptualized "Farmer Trust Score" to aggregate government verification, past project success, and peer reviews into a single, immutable credibility rating.

Smart Task & Productivity Manager — Java Desktop Application

Module: Individual Java Project | Full-Stack CLI Application

Overview: A robust Java application that allows users to manage tasks, track deadlines, and analyze productivity. Unlike simple CLI apps, it persists data using a MySQL database. The application follows a clean, layered architecture, demonstrating Separation of Concerns: Presentation (CLI), Service (Business Logic), DAO (Database Access), and Model (Task Object).

Technologies Used

  • Java 8+ | Object-Oriented Programming (Encapsulation, Collections, Streams API)
  • JDBC & MySQL | PreparedStatement to prevent SQL Injection
  • Maven | Dependency Management (MySQL Connector)
  • CLI User Interface with Input Validation & Exception Handling
  • Sorting & Filtering using Java Streams | Productivity Calculations

Key Features

  • CRUD Operations: Create, Read, Update, Delete tasks
  • Persistent Storage: Tasks saved in MySQL even after closing the app
  • Smart Sorting: Sort tasks by due date instantly
  • Filtering: View tasks by priority levels using Java Streams
  • Productivity Report: Calculates task completion percentage dynamically
  • Input Validation & Exception Handling: Prevents invalid data entries and ensures stability

Skills Gained

  • Layered Architecture & Clean Code Principles
  • SQL Database Design & JDBC Integration
  • Advanced Java Collections & Streams API
  • CLI Application Design & User Input Validation
  • Exception Handling & Defensive Programming

Challenges & Problem-Solving

  • Maintaining persistent storage across multiple layers without mixing SQL logic in the CLI
  • Sorting and filtering tasks efficiently using Java Streams
  • Ensuring accurate productivity calculations and preventing invalid date/priority entries
